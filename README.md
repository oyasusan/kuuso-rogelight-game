# 100秒アイドルライブ

**100秒ローグライト × アイドルライブアクション**

「敵を倒すゲームではなく、100秒で最高のライブを完成させるゲーム」

プレイヤーはアイドルとなり、100秒間ライブを行います。観客の熱量（Heat）を上げて熱狂させると、熱狂が周囲の観客へ連鎖し、会場全体が盛り上がっていきます。ライブ終了時の盛り上がりがスコアになります。

## セットアップ

Node.js 20 以上が必要です。

```bash
npm install      # 依存パッケージのインストール
npm run dev      # 開発サーバー起動（表示された URL をブラウザで開く）
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果の確認
```

| 操作 | キー |

## ゲーム仕様

### 基本ルール

- ライブは 100 秒間。終了時にリザルトを表示する
- プレイヤーは画面内を自由に移動できる
- 2 秒ごとに自動で「歌」を発動し、半径 120px 以内の観客の Heat を +20 する（音符エフェクト付き）

### 観客と Heat

- 観客は 112 人（16 × 7 の格子＋ランダムなずらし）を配置
- 各観客は Heat（0〜100、初期値 20）と Mood（感情、将来拡張用）を持つ
- Heat は色で表現される（青 → 赤、熱狂時は黄色）

### 熱狂状態と連鎖

- Heat が 100 に達すると **熱狂状態** になる
- 熱狂状態の観客は毎秒、周囲 100px 以内の観客へ Heat +5 を与える
- → 盛り上がりが会場全体へ連鎖していく（このゲームの核となる体験）

### レベルと経験値

- 観客が 1 人熱狂するごとに経験値 +10
- 一定経験値でレベルアップ（Phase1 では通知のみ。強化 3 択は Phase2 で実装）

### コンボ

- 観客が熱狂するたびにコンボ +1
- 3 秒間新しい熱狂が発生しないとコンボが途切れる
- 最大コンボはリザルトに表示される

### スコア

```
最終スコア = 平均Heat × 100 + 熱狂人数 × 30 + 最大コンボ × 15
```

### UI（画面上部）

残り時間 / レベル / 経験値 / 平均 Heat / 熱狂人数（コンボは右上に表示）

## ディレクトリ構成

```
├── index.html            エントリー HTML
├── vite.config.js        Vite 設定
└── src/
    ├── main.js           Phaser 起動設定
    ├── constants.js      全定数（バランス調整はここ）
    ├── scenes/
    │   ├── HomeScene.js      タイトル画面
    │   ├── GameScene.js      ライブ本編（進行・タイマー・コンボ）
    │   └── ResultScene.js    リザルト画面
    ├── objects/
    │   ├── Player.js         プレイヤー（アイドル）
    │   ├── Audience.js       観客（Heat / Mood / 熱狂状態）
    │   ├── Performance.js    パフォーマンス基底クラス＋歌
    │   └── NoteEffect.js     音符エフェクト（オブジェクトプール）
    ├── systems/
    │   ├── HeatSystem.js     Heat の一元管理・熱狂の連鎖
    │   ├── SpawnSystem.js    観客の生成
    │   └── LevelSystem.js    経験値・レベル管理
    ├── ui/
    │   └── HUD.js            画面上部の情報表示
    └── assets/               画像・音声素材（現状はテクスチャを動的生成）
```

※ `AntiFan.js` / `UpgradeSystem.js` / `UpgradePanel.js` は Phase2 で追加予定。

## 実装フェーズ

- [x] **Phase1（実装済み）**: プレイヤー移動 / 観客生成 / Heat システム（熱狂の連鎖含む） / 自動パフォーマンス（歌） / 100 秒タイマー / リザルト
- [ ] **Phase2**: アンチ / レベルアップ 3 択 / スキル追加（ダンス・ファンサ・MC）
- [ ] **Phase3**: 演出（Heat 平均値による会場演出の変化） / エフェクト / SE / BGM
- [ ] **Phase4**: 永久強化 / キャラクター追加 / 新ステージ

## 技術構成

- HTML5 / JavaScript (ES6 modules)
- Phaser 3（Canvas / WebGL 描画、60FPS）
- Vite（開発サーバー・ビルド）
- エフェクトはオブジェクトプーリングで再利用
- `Scale.FIT` によるレスポンシブ対応（将来のスマホ対応を想定）
