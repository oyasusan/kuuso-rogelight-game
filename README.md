# 100秒ロマンス

**100秒ローグライト × アイドルライブアクション**

「敵を倒すゲームではなく、100秒で最高のライブを完成させるゲーム」

プレイヤーはアイドルとなり、100秒間ライブを行います。観客の熱量（Heat）を上げて熱狂させると、熱狂が周囲の観客へ連鎖し、会場全体が盛り上がっていきます。ライブ終了時の盛り上がりがスコアになります。

## セットアップ

Node.js 20 以上が必要です。

```bash
npm install      # 依存パッケージのインストール
npm run dev      # 開発サーバー起動（表示された URL をブラウザで開く）
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果の確認
```

## 操作方法

| 操作 | キー |
| --- | --- |
| 移動 | WASD / 矢印キー |
| 開始・タイトルへ戻る | スペースキー / クリック |

攻撃ボタンはありません。パフォーマンスは自動で発動します。

## ゲーム仕様

### 基本ルール

- ライブは 100 秒間。終了時にリザルトを表示する
- プレイヤーは画面内を自由に移動できる
- パフォーマンスはすべて自動発動

### パフォーマンス

| 名前 | 取得 | 効果 |
| --- | --- | --- |
| 歌 | 最初から | 2 秒ごとに半径 120px 内の観客へ Heat +12、アンチへダメージ 1 |
| ダンス | レベルアップ | 3 秒ごとに前方の扇形範囲（半径 150px・100°）へ Heat +30 |
| ファンサ | レベルアップ | 5 秒ごとに最も近い観客を即座に熱狂状態にする |
| MC | レベルアップ | 20 秒ごとに発動し、10 秒間すべての Heat 上昇量を 1.5 倍にする |

ダンスの扇形は最後に移動した方向を向きます。

### アンチ

- ライブ開始 12 秒後から、8 秒間隔で画面外からスポーンしてプレイヤーへ近づく（時間経過でスポーン数が増える）
- プレイヤーに接触すると **未熱狂の観客全体の Heat -10**
- さらに **熱狂状態の観客も 20%（最低 1 人）が熱狂を解除され、Heat 50 まで冷める**。積み上げた盛り上がりも巻き戻るので、避け続ける・早めに歌で倒すことに意味がある
- 歌のダメージを 3 回受けると退場する
- アンチはオブジェクトプールで管理される

### 観客と Heat

- 観客は 112 人（ステージごとの格子配置＋ランダムなずらし）を配置
- 各観客は Heat（0〜100、初期値 20）と Mood（感情、将来拡張用）を持つ
- **Heat は毎秒 3 ずつ自然に冷めていく**。盛り上げ続けないと会場は静まる
- Heat は色（青 → 赤、熱狂時は黄色）と動きで表現される。冷めているとほぼ静止し、温まるほど大きく揺れ、熱狂するとジャンプする

### 熱狂状態と連鎖

- Heat が 100 に達すると **熱狂状態** になる（熱狂した観客は自然減衰しない。ただしアンチの妨害では解除されうる）
- 熱狂状態の観客は毎秒、周囲 100px 以内の観客へ Heat +6 を与える
- **1 人の対象が 1 tick に受け取る上昇量は、周囲に熱狂源が何人いても +6 の 1 回分だけ**（単純合算にすると熱狂の密度が上がるほど雪だるま式に加速してしまうため、対象ごとに上限を設けている）
- → 盛り上がりが会場全体へ連鎖していく（このゲームの核となる体験）が、急激な加速はせず、終盤にかけて少しずつ会場全体が熱狂へ近づいていく

### レベルと経験値

- 観客が 1 人熱狂するごとに経験値 +10
- 一定経験値でレベルアップすると **ゲームが停止し、強化 3 択** が表示される（クリックまたは 1〜3 キーで選択）
- 必要熱狂人数（累計）: Lv2=10 人 / Lv3=24 人 / Lv4=44 人 / Lv5=71 人 / Lv6=110 人。ほぼ全員を熱狂させると終盤に Lv6（実質最大）へ届く

### アップグレード（ランダム 3 択）

- 歌強化（Heat +6、ダメージ +1）
- ダンス取得 / 強化（Heat +10）
- ファンサ取得 / 強化（クールタイム -0.5 秒、下限 2 秒）
- MC 追加 / 強化（効果時間 +2 秒）
- パフォーマンス範囲 UP（歌・ダンスの半径 15% 拡大）
- 移動速度 UP（+12%）

### コンボ

- 観客が熱狂するたびにコンボ +1
- 3 秒間新しい熱狂が発生しないとコンボが途切れる
- 最大コンボはリザルトに表示される

### 会場演出（平均 Heat で変化）

| 平均 Heat | 演出 |
| --- | --- |
| 〜39 | 静か |
| 40〜 | 少し歓声（歓声の環境音が鳴り始める） |
| 60〜 | サイリウム（熱狂した観客が頭上でサイリウムを振る） |
| 80〜 | 紙吹雪が舞う |
| 100 | レーザー演出・歓声最大 |

段階が上がると画面中央にアナウンスが表示されます。上位の段階は下位の演出を含み、平均 Heat が下がると演出も収まります。

### サウンド

音声アセットは使わず、すべて Web Audio API でプロシージャル生成しています。

- BGM: ライブ中の 8 ビートループ（キック・ハイハット・ベース）
- 歓声: 演出段階に応じて音量が変わる環境ノイズ
- SE: 熱狂・レベルアップ・アップグレード選択・アンチ命中 / 接触・ライブ開始・リザルトのファンファーレ

※ ブラウザの自動再生制限のため、音はタイトル画面で操作した後から鳴ります。

### スコア

```
最終スコア = 平均Heat × 100 + 熱狂人数 × 30 + 最大コンボ × 15
```

### キャラクター（タイトルで選択）

プレイヤーは画像アセットを使わず、楕円・台形の数式からドット絵を生成して表示している（観客・アンチと同じくプロシージャル生成）。髪・襟・ヘム・ブーツにキャラクターの差し色を使い、肌・衣装ベースは共通にすることで、少ないコードで見分けやすい 3 人を表現している。

| 名前 | タイプ | 特徴 |
| --- | --- | --- |
| アイカ | バランス型 | 標準性能 |
| ミオ | 歌姫型 | 歌の Heat +30% / 移動速度 -10% |
| リン | スピード型 | 移動速度 +25% / 歌の範囲 -15% |

### ステージ（タイトルで選択）

| 名前 | 特徴 |
| --- | --- |
| ライブハウス | 標準的な会場（観客 112 人・1 ブロック） |
| 野外フェス | 観客が 2 ブロックに分かれ、アンチの出現間隔が 30% 短い |

### 永久強化（メタ進行）

- ライブ終了時に `スコア ÷ 100` 人の **ファン** を獲得する
- リザルト後の永久強化画面（タイトルからも遷移可能）でファンを消費して購入する
- 強化はラン間で持ち越され、localStorage に保存される

| 強化 | 効果（1 ランクごと） | 最大 | 費用 |
| --- | --- | --- | --- |
| ファンの期待 | 観客の初期 Heat +5 | 4 | 100×(ランク+1) |
| ボイストレーニング | 歌の Heat +4 | 5 | 80×(ランク+1) |
| 体力づくり | 移動速度 +8% | 5 | 80×(ランク+1) |
| メンタルケア | アンチの Heat 減少を 2 緩和 | 4 | 120×(ランク+1) |

### 設定

タイトル右上の「設定」から、**永久強化のリセット**ができます（所持ファン・全ランクを 0 に戻す。キャラクター・ステージの選択は保持）。取り消せない操作のため、実行前に確認ダイアログが挟まります。

### UI（画面上部）

残り時間 / レベル / 経験値 / 平均 Heat / 熱狂人数（コンボは右上に表示）

## ディレクトリ構成

```
├── index.html            エントリー HTML
├── vite.config.js        Vite 設定
└── src/
    ├── main.js           Phaser 起動設定
    ├── constants.js      全定数（バランス調整はここ）
    ├── scenes/
    │   ├── HomeScene.js      タイトル画面（キャラクター・ステージ選択）
    │   ├── GameScene.js      ライブ本編（進行・タイマー・コンボ）
    │   ├── ResultScene.js    リザルト画面（ファン獲得）
    │   ├── PermanentUpgradeScene.js  永久強化画面
    │   └── SettingsScene.js  設定画面（永久強化のリセット）
    ├── objects/
    │   ├── Player.js         プレイヤー（アイドル）
    │   ├── Audience.js       観客（Heat / Mood / 熱狂状態）
    │   ├── AntiFan.js        アンチ（妨害キャラクター）
    │   ├── Performance.js    パフォーマンス（歌・ダンス・ファンサ・MC）
    │   ├── NoteEffect.js     音符エフェクト（オブジェクトプール）
    │   ├── idolPixelGrid.js  ドット絵アイドルの形状データ（Phaser 非依存）
    │   └── PixelArt.js       ドット絵アイドルのテクスチャ生成（Phaser Graphics）
    ├── systems/
    │   ├── HeatSystem.js     Heat の一元管理・熱狂の連鎖
    │   ├── SpawnSystem.js    観客・アンチの生成（アンチはプール管理）
    │   ├── LevelSystem.js    経験値・レベル管理
    │   ├── UpgradeSystem.js  アップグレードの抽選と適用
    │   ├── StageDirector.js  会場演出（サイリウム・紙吹雪・レーザー）
    │   ├── AudioSystem.js    Web Audio による BGM・SE・歓声
    │   ├── SaveSystem.js     セーブデータ（localStorage）
    │   └── RunModifiers.js   キャラ・ステージ・永久強化のラン設定への反映
    ├── ui/
    │   ├── HUD.js            画面上部の情報表示
    │   └── UpgradePanel.js   レベルアップ時の 3 択パネル
    └── assets/               画像・音声素材（現状はテクスチャを動的生成）
```
